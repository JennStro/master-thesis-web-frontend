Equals operator and equals method explanation

In Python, you can write:

    [1, 2, 3] == [1, 2, 3]

and it would be true, because Python calls the objects equals-method when you use ==. In Java on the other hand, this would not be true. When you use == in Java, it checks that the two objects are of the same instance (they have the same location in memory). For example

    Bag bag = new Bag();
    System.out.println(bag == bag);

this would be true, because bag are of the same instance and have the same memory location. However, if you try to do

    Bag bag1 = new Bag();
    Bag bag2 = new Bag();
    System.out.println(bag1 == bag2);

this would be false because bag1 and bag2 are different instances. To compare two objects, we need to use the equals method: 

    Bag bag1 = new Bag();
    Bag bag2 = new Bag();
    System.out.println(bag1.equals(bag2));

but if we do not implement the equals method in class Bag, this too will not work, we still compare memory locations! This is because every object in Java inherits from the type Object. Object's equals method compare memory locations, so Bag will do this as well. So, for the code above to work, we have to implement the equals method in the class Bag. Below is an example of how the equals method for Bag could look like: 

    @Override
    public boolean equals(Object o) {

        if (this == o) return true; //If two objects have the same memory location, they definetly are the same!

        if (o == null || getClass() != o.getClass()) return false; //If one object is null or they have different classnames they are not the same.

        Bag bag = (Bag) o; //We promise that this is a Bag object.

        return size == bag.size; //Two bags are equal if they have the same size.
    }

So, when should you use == in Java?

  • We use == to compare the primitive types in Java! (int, boolean, double, long...), because these are not objects.
  • When checking if an object is null, you need to use == (To avoid NullPointerException).
