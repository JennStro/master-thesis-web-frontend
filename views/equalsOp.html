Equals operator explanation

In Python, you can write:

    [1, 2, 3] == [1, 2, 3]

and it would be true. In Java on the other hand, this would not be true. When you use == in Java, it checks that the two objects are of the same instance. For example

    Bag bag = new Bag();
    System.out.println(bag == bag);

this would be true, because bag are of the same instance. However, if you try to do

    Bag bag1 = new Bag();
    Bag bag2 = new Bag();
    System.out.println(bag1 == bag2);

this would be false because bag1 and bag2 are different instances. To check if bag1 and bag2 is equal, we have to implement the equals method in the class Bag and define what makes a bag equal (maybe its contents?) and use the equals method. We have to override the equals method, because by default it uses the Object equals method, that uses == to check for equality. 

    Bag bag1 = new Bag();
    Bag bag2 = new Bag();
    System.out.println(bag1.equals(bag2));

Assuming we have an equals method in Bag that compares the bag contents, this would be true because both bags are empty.
